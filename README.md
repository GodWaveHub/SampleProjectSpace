# MyProjectSpace

GitHub Copilotを活用したシステム開発プロジェクト管理ワークスペース

---

## 📋 プロジェクト概要

このリポジトリは、GitHub Copilotを活用してシステム開発プロジェクトを効率的に管理するためのワークスペースです。見積からWBS作成、スケジュール管理、課題管理、進捗管理、会議管理、設計、実装、テストまで、プロジェクト全体のライフサイクルを一元管理します。

### 📊 01_Estimate（見積）
プロジェクトの見積に関する資料を格納します。
- **用途**: プロジェクト規模の算出、工数見積、コスト見積
- **格納ファイル例**: 見積書、機能一覧、規模算出表、コスト計算書、見積前提条件一覧  

### 📋 02_WBS（Work Breakdown Structure）
作業分解構造とタスク定義を管理します。
- **用途**: プロジェクトの作業分解、タスクの詳細定義
- **格納ファイル例**: WBS表、タスク定義書、作業項目一覧
- **活用方法**: 見積結果を基にした詳細なタスク分解

### 📅 03_Schedule（スケジュール）
プロジェクトのスケジュール管理を行います。
- **用途**: プロジェクト全体のスケジュール管理、マイルストーン設定
- **格納ファイル例**: ガントチャート、スケジュール表、マイルストーン計画
- **活用方法**: WBSを基にしたスケジュール作成と進捗追跡

### ⚠️ 04_Issues（課題管理）
プロジェクトの課題とリスクを管理します。
- **用途**: 課題の識別、追跡、解決状況の管理
- **格納ファイル例**: 課題管理表、リスク管理表、対応策一覧
- **活用方法**: 定期的な課題レビューと対策検討

### 📈 05_Progress（進捗管理）
プロジェクトの進捗状況を追跡・管理します。
- **用途**: 週次・月次の進捗報告、パフォーマンス分析
- **格納ファイル例**: 進捗報告書、実績管理表、パフォーマンス分析
- **活用方法**: スケジュールとの対比による進捗評価

### 🤝 06_Meetings（会議・議事録）
プロジェクト関連の会議資料と議事録を管理します。
- **用途**: 会議の準備、実施、フォローアップ
- **格納ファイル例**: 会議資料、議事録、アクションアイテム
- **活用方法**: 定期的なプロジェクトレビューと意思決定の記録

### 🎨 07_Design（設計）
システムの設計書を工程別に管理します。

#### 07_Design/01_BasicDesign（基本設計）
- **用途**: システムの基本的な設計仕様の定義
- **格納ファイル例**: 基本設計書、システム構成図、画面遷移図
- **活用方法**: 要件定義を基にした高レベル設計

#### 07_Design/02_DetailDesign（詳細設計）
- **用途**: 実装レベルの詳細な設計仕様の定義
- **格納ファイル例**: 詳細設計書、データベース設計書、API仕様書
- **活用方法**: 基本設計を基にした実装可能レベルの設計

#### 07_Design/03_DatabaseDesign（データベース設計）
- **用途**: データベースの物理設計と論理設計の定義
- **格納ファイル例**: ER図、テーブル定義書、インデックス設計書
- **活用方法**: 詳細設計に基づいたデータベース構造の最適化

### 💻 08_Project（実装）
- **用途**: プログラム開発のルートフォルダ
- **格納ファイル例**: プログラムコードやテストコード、設定ファイルなど
- **活用方法**: 実行モジュールの開発

### 🧪 09_Test（テスト）
各段階のテスト計画、実施、結果を管理します。

#### 09_Test/01_UnitTest（単体テスト）
- **用途**: 個別モジュールのテスト
- **格納ファイル例**: 単体テスト仕様書、テストケース、テスト結果
- **活用方法**: 実装と並行したテスト実施

#### 09_Test/02_IntegrationTest（結合テスト）
- **用途**: モジュール間の連携テスト
- **格納ファイル例**: 結合テスト仕様書、テストケース、テスト結果
- **活用方法**: 単体テスト完了後の統合検証

#### 09_Test/03_SystemTest（システムテスト）
- **用途**: システム全体の総合テスト
- **格納ファイル例**: システムテスト仕様書、テストケース、テスト結果
- **活用方法**: 結合テスト完了後の最終検証
